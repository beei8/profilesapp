## ­čÜÇ Deployment Steps

### 1. Push to GitHub
```bash
git add .
git commit -m "Ready for Render.com deployment"
git push origin main
```

### 2. Create PostgreSQL Database on Render
1. Go to [render.com](https://render.com)
2. Click **New +** Ôćĺ **PostgreSQL**
3. Configure:
   - **Name:** `profilesapp-db`
   - **Database:** `profilesapp`
   - **User:** `postgres`
   - **Plan:** Free
4. Copy the **Internal Database URL**

### 3. Deploy Web Service
1. Click **New +** Ôćĺ **Web Service**
2. Connect GitHub repository
3. Configure:
   - **Name:** `profilesapp-server`
   - **Plan:** Free
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`

### 4. Set Environment Variables
Add to Web Service > Environment:

```
NODE_ENV=production
JWT_SECRET=<auto-generated>
SETUP_KEY=<auto-generated>
DB_USER=postgres
DB_PASSWORD=<from-database-dashboard>
DB_HOST=<from-database-internal-url>
DB_PORT=5432
DB_NAME=profilesapp
PORT=<leave-blank-render-assigns>
```

### 5. Deploy
Click **Create Web Service** and wait for deployment

### 6. Create Admin User
```bash
curl -X POST https://your-service.onrender.com/api/setup \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "YourSecurePassword123!",
    "setupKey": "<your-SETUP_KEY-value>"
  }'
```

## ÔÜá´ŞĆ Important Notes

### Database Limits
- **Free tier:** 90-day auto-deletion
- **Solution:** Use paid tier for production
- **Backup:** Export data before 90 days if using free tier

### Service Limits
- **Free tier:** Spins down after 15 min inactivity
- **Solution:** Use paid tier for always-on service

### Common Issues

ÔŁî **Connection refused**
- Verify database is running
- Check credentials in environment variables
- Ensure network access is allowed

ÔŁî **Health check failing**
- Database may not be initialized
- Check logs: Render > Service > Logs
- Verify `DB_HOST`, `DB_USER`, `DB_PASSWORD`

ÔŁî **Cannot create admin user**
- Verify `SETUP_KEY` matches environment variable
- Ensure database tables were created
- Check `/health` endpoint responds
